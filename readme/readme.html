1. ¿Por qué evitar los callbacks en las llamadas asíncronas?

Porque vuelven el código más complejo y puede resultar difícil de entender. Antes se usaba para desarrollar operaciones asíncronas, pero luego con el uso de las promesas ya se simplificó la situación.

2. ¿Qué es una promesa?

Es un objeto que determina el status de una operación. Por ejemplo, la promesa puede estar pendiente, en un inicio, puede ser resulta o, en todo caso, puede ser rechazada. También puede suceder que durante la operación ocurra un error. Las promesas se usan para operaciones asíncronas; es decir, cuando el código no necesariamente se lee de manera sincrónica (una línea debajo de otra línea). Las promesas también se utilizan para el manejo de información, con los fetch.

3. ¿Qué es el callback hell?

Es cuando se ejecutan varios callbacks que están anidados. Por ejemplo, cuando en el navegador aparece información y en determinado tiempo aparece una nueva información y así sucesivamente, pero todo dentro de una misma función.
